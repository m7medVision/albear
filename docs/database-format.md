# albear — Database Format

SQLite, single writer (`vaultd` only), WAL + FULL synchronous, foreign keys
on, STRICT tables, trusted_schema off, prepared statements only. Every write
runs in a transaction. Migrations are embedded, ordered, and checksummed
(SHA-256); a checksum mismatch refuses startup. Downgrade is unsupported;
newer format versions refuse to load (fail closed).

Location (not secret — contents are ciphertext):

```text
$XDG_DATA_HOME/albear/vault.db        0600, directory 0700
$XDG_CONFIG_HOME/albear/daemon.key    0600 (transport identity key)
$XDG_RUNTIME_DIR/albear/vault.sock    0600, directory 0700
```

## Tables (migration 0001)

- `schema_migrations(version, applied_at_ms, checksum)` — managed by the migrator.
- `vault` — singleton row: vault_id (16B), format_version, active_envelope_version. No secrets.
- `key_envelopes` — Argon2id parameters + salt, wrapped root key
  (XChaCha20-Poly1305 under the KEK), encrypted canary. Replaced atomically on
  master-password change (insert new → activate → delete old, one transaction).
- `records` — per record: random 16B id, key_version (AAD tag of the root-key
  generation; deliberately NOT a foreign key to key_envelopes — password
  changes replace envelopes without touching records), revision (optimistic
  concurrency via `UPDATE … WHERE revision = ?`), independent
  nonce+ciphertext pairs for metadata and secret payloads. No plaintext
  title/username/URL/tag/timestamp anywhere.
- `clients` — kind, status, capability mask, credential verifier
  (SHA-256 of the credential; doubles as the Noise PSK), pinned Noise static
  public key, encrypted label.
- `security_events` — numeric code + severity; details encrypted under the
  audit key, or absent when recorded while locked.

## CQRS access (sqlc)

All SQL is simple single-statement queries generated by sqlc, split by
command/query responsibility:

```text
internal/infrastructure/sqlite/sql/commands.sql → gen/command  (INSERT/UPDATE/DELETE only)
internal/infrastructure/sqlite/sql/queries.sql  → gen/query    (SELECT only)
```

`Store.Command(ctx, fn)` wraps the write side in a transaction;
`Store.Query()` serves reads. No hand-written SQL outside those two files
except the migrator's own bookkeeping.

## Backups

`VACUUM INTO` snapshot inside a versioned container:

```text
"ALBEARBK" ‖ version(u32) ‖ vault_id(16B) ‖ created_ms(u64) ‖ len(u64) ‖ snapshot ‖ HMAC-SHA-256
```

HMAC key = backup subkey (HKDF of the root key), so verification requires an
unlocked vault; format checks work locked. Restore verifies, moves the live
database to `vault.db.recovery`, swaps atomically, re-migrates, and leaves the
vault locked.
